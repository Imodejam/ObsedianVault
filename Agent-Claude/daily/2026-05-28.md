# 2026-05-28

## Puntify — link condivisi booking senza prefisso lingua
- `BookingOptions.razor` (App merchant): `PublicBookingUrl` (`/m/{slug}`) e `DirectBookingUrl` (`/book/{slug}`) generavano l'URL con prefisso `/{CurrentLang}/`.
- Stefano: il link da condividere non deve contenere la lingua → lingua automatica.
- Fix: rimosso `/{CurrentLang}/` e la proprietà `CurrentLang` (dead code). Il middleware Vetrina (Program.cs) già redirige `/m/*` e `/book/*` senza prefisso alla lingua del browser (Accept-Language).
- Build App OK (0 errori), restart `puntify-app.service` → active.

## Puntify — orari Agenda non devono mostrare orari operatori
- Stefano: configurando gli orari dell'agenda comparivano anche gli orari degli operatori.
- Causa: tabella `booking_availability` contiene sia righe PV (`operator_id` NULL) sia righe operatore (`operator_id` valorizzato).
  - **Load**: `BookingSchedule.razor` filtrava solo per `shop_id` → pescava anche righe operatore. Fix: skip righe con `OperatorId != null`.
  - **Save (bug correlato grave)**: `BookingController.SaveAvailability` faceva `DeleteAsync("booking_availability","shop_id=eq.{shopId}")` → cancellava ANCHE la disponibilità degli operatori a ogni salvataggio agenda. Fix: filtro `&operator_id=is.null` (pattern già usato in TableBookingController:231).
- Build App + Server OK (0 errori). Restart puntify-app + puntify-server.
- NOTA infra: i servizi girano con `dotnet watch`; il restart via systemctl manda in timeout lo stop del vecchio processo (SIGKILL) → stato "failed". Workaround: `systemctl reset-failed` + `start`. In dev il watch ricompila comunque al salvataggio del file.
