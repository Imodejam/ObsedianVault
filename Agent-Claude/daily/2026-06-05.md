## Puntify · Fix login persistente (msg 3346) FATTO+DEPLOY
- Causa: SupabaseOptions senza SessionHandler/PersistSession → a ogni reload (WASM) la libreria parte senza sessione e si affidava solo al cookie custom `punto_account`, fragile con la rotazione dei refresh token (e a rischio limite 4KB) → login ogni volta.
- Fix: nuovo `LocalStorageSessionPersistence : IGotrueSessionPersistence<Session>` (sync via IJSInProcessRuntime, WASM) su localStorage; impostato `options.SessionHandler` in Program.cs. Con AutoRefreshToken=true la libreria ricarica la sessione all avvio e rinnova l access token trasparente; SaveSession scrive sempre l ultimo refresh token ruotato. Aggiunto Newtonsoft.Json 13.0.3 per serializzare Session. InitializeAsync ora in try/catch (token morto non blocca l app).
