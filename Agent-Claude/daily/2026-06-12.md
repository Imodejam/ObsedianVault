## msg 3543 — Mega-menu Funzionalità: freccia "tuttuno" col pannello + pannello sopra banner app (FATTO+DEPLOY)
1) Freccia staccata dal pannello (gap visibile). Causa: arrow era position:absolute ancorata al trigger (top:100%+margin), il pannello è position:fixed top:4.6rem → riferimenti diversi, gap variabile con altezza header/scroll. Fix (app.css .mega-arrow): position:fixed top:calc(4.6rem - 8px), 16px → il rombo straddla il bordo superiore del pannello. Stessa logica fixed del pannello → stesso containing block (viewport, o header quando .glass ha backdrop-filter; header è a top:0 quindi 4.6rem≈viewport in entrambi) → saldata in ogni stato di scroll. X = static position (sotto il trigger).
2) Pannello deve stare sopra il banner "Installa Puntify" (PwaInstallBanner fixed bottom-0 z-50). Causa: header è fixed z-50 → crea stacking context, il pannello (z-60) è capped a z-50 dell'header = pari al banner, e il banner è DOPO l'header nel DOM (MainLayout) → vinceva. Fix: HeaderClass z-50 → z-[60] (Header.razor) → tutto l'header (pannello incluso) sopra il banner. Bonus: anche drawer mobile ora sopra il banner.
CSS bump: app.css?v=20260603b → 20260612a (App.razor). Build 0 err, vetrina riavviata, /it HTTP 200, css v + z-[60] nel render. Verifica visiva freccia: a Stefano (hover desktop, non curl-abile).
